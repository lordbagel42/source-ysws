## 2026-03-23 - Optimizing DB Connection Pooling and Image Loading
**Learning:** In a Cloudflare Workers environment using Hyperdrive, creating a new `postgres-js` client and Drizzle instance on every request (or worse, every property access via a Proxy) significantly increases latency and risks exhausting database connections. Caching these instances in a global `Map` keyed by the connection string ensures proper connection pooling within the same worker isolate. Additionally, frontend images from external providers like Unsplash should always use `auto=format` and appropriate width constraints to reduce payload size.
**Action:** Always implement a module-level cache for database clients in server-side logic and ensure all external images are served with modern formats and lazy loading.
