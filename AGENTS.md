# Project Overview

**source-ysws** is a SvelteKit web application deployed to Cloudflare Workers. It uses GitHub OAuth for authentication and PostgreSQL for data storage.

## Technology Stack

- **Framework**: SvelteKit 2 with Svelte 5 (runes/`$props()`)
- **Language**: TypeScript (strict mode)
- **Styling**: Tailwind CSS v4 (with `@tailwindcss/forms` and `@tailwindcss/typography` plugins)
- **Database**: PostgreSQL via Drizzle ORM
- **Authentication**: better-auth with GitHub OAuth provider
- **Markdown**: mdsvex (`.svx` files supported alongside `.svelte`)
- **Deployment**: Cloudflare Workers via `@sveltejs/adapter-cloudflare` + Wrangler
- **Package Manager**: pnpm (engine-strict enabled)
- **Testing**: Playwright (e2e)
- **Linting/Formatting**: ESLint + Prettier (tabs, single quotes, no trailing commas)

# Essential Commands

All commands use **pnpm**. Do not use npm or yarn.

```sh
# Install dependencies
pnpm install

# Start dev server
pnpm dev

# Type-check
pnpm check

# Lint (Prettier + ESLint)
pnpm lint

# Auto-format
pnpm format

# Build for production
pnpm build

# Preview production build (via Wrangler)
pnpm preview

# Deploy to Cloudflare
pnpm deploy

# Run e2e tests (builds first, then runs Playwright)
pnpm test

# Database commands (require DATABASE_URL in env)
pnpm db:start      # Start local PostgreSQL via Docker
pnpm db:push        # Push schema to database (dev)
pnpm db:generate    # Generate migration files
pnpm db:migrate     # Run migrations
pnpm db:studio      # Open Drizzle Studio GUI

# Regenerate auth schema after changing auth config
pnpm auth:schema

# Regenerate Cloudflare worker types
pnpm cf-typegen
```

# Patterns

## Authentication

Session data is attached in `src/hooks.server.ts` via `event.locals.session` and `event.locals.user`. Access these in `+page.server.ts` load functions and form actions.

## Database Migrations

1. Edit `src/lib/server/db/schema.ts`.
2. Run `pnpm db:generate` to create a migration.
3. Run `pnpm db:migrate` to apply it.
4. If you change auth config in `src/lib/server/auth.ts`, also run `pnpm auth:schema` to regenerate `auth.schema.ts`.

## Svelte 5

This project uses **Svelte 5 runes**. Use `$props()`, `$state()`, `$derived()`, `$effect()` — not legacy `export let` or `$:` reactive statements.

# Anti-Patterns

- **Do NOT edit `src/lib/server/db/auth.schema.ts` by hand.** It is auto-generated by `pnpm auth:schema`.
- **Do NOT use npm or yarn.** The project enforces pnpm (`engine-strict=true` in `.npmrc`).
- **Do NOT use Svelte 4 syntax** (`export let`, `$:` reactive declarations, `on:click`). Use Svelte 5 runes and `onclick` attributes.

# Code Style

- **Tabs** for indentation (not spaces).
- **Single quotes**, no trailing commas.
- **Print width**: 100 characters.
- Prettier and ESLint configs are authoritative — run `pnpm format` then `pnpm lint` before committing.
- Svelte files are parsed with `prettier-plugin-svelte`; Tailwind classes are sorted by `prettier-plugin-tailwindcss`.
